Typography

There are two main fonts on the site, Montseratt and Lato. Montseratt is to only be used for headings and buttons. Lato is the global body font and can be used anywhere except headings.

Font Calculation

Font sizes for headings and other elements are calculated using REMs.

Font sizes are reset to a base number that when multplied by 10 displays the same for both EMs and pixels.

Each heading has a different font-size for the different viewports it will be viewed on to maxmize readbility.

.font-size(@sizeValue) {
  	@remValue: @sizeValue;
  	@pxValue: (@sizeValue * 10);
  	font-size: ~"@{pxValue}px"; 
  	font-size: ~"@{remValue}rem";
}

.line-height(@lineValue) {
	@emValue: @lineValue;  
	@pxValue: (@lineValue * 10);
	line-height: ~"@{pxValue}px";
	line-height: ~"@{emValue}rem";
}

Heading 1

h1 {.font-size(4); .line-height(5.2);
	@media @mobile {.font-size(3);}
	@media @tablet {.font-size(3.5);} 
	a {color: @dk-gray;}
}

Buttons

Buttons require two classes to achieve different styles. The first class must be applied to all buttons, btn makes the button look like a button. The second class gives it a color to give it context: btn-default, btn-primary, btn-default, btn-warning, btn-danger, & btn-light.

There are additional styles that you can use with buttons.

  • mont Makes the button use Montseratt as the font and makes the text uppercase.
  • btn-block Makes the button span the entire width of it's continer by making it a block level element.

Default

A light gray button that can be used for generic site actions.

<button class="btn btn-default">
   Default Regular
</button>

Default size

<button class="btn btn-default btn-sm">
   Default Small
</button>

Make a button smaller by adding btn-sm to the button class.

<button class="btn btn-default btn-lg">
   Default Large
</button>

Make a button larger by adding btn-lg to the button class.

<button class="btn btn-default mont">
   Default Montseratt
</button>

Change the font to Montseratt and make it uppercase by adding mont to the button class.

Product Feature Modules

To highlight the product features, you can use a variety of layouts based on what the feature is.

Each feature has a headline contained in a h3 and a short description (140 characters or less), contained in a p

Remove all text within the double brakets [[ ]] with the feature headline, feature copy, or product image. When inserting a URL make sure that you are using the absolute path to the image. Make sure that http:// is at the beginning of the URL.

Product Feature with image on the right

Sometimes the photo just looks better on the right!

Insert this entire code block where you would like this feature style to appear:

<div class="container">
	<div class="row">
		<div class="col-sm-6 col-md-5 col-md-offset-1 feature-description">
			<h3>[[FEATURE HEADLINE]]</h3>
			<p>[[FEATURE COPY]]</p>
			<h3>[[FEATURE HEADLINE]]™</h3>
			<p>[[FEATURE COPY</p>
		</div>

		<div class="col-sm-6 col-md-5 feature-img">
			<img src="[[IMAGE URL]]" class="img-responsive" />
		</div>
	</div>
</div>

Product Images

Users should see beautiful product images when browsing the marketplace. Product feature images and hero images should be as high res as possible, while still creating a pleasent experience with fast loading images.

Image size

  • Image size should be minimum 520x520 pixels
  • Image must include white background
  • Image cannot be transparent
  • Image content should be centered on white background
  • Image content must be a minimum of 500 pixels on the widest edge (height or width)
  • Image content must have a 10px gutter around it.

Product feature hero image size

  • Image size should be minimum 1600x500 pixels
  • Image cannot be transparent
  • Image subject should be either left or right justified, never in the center.

Supported Screen Sizes

We are only supporting the following screen sizes at launch for Marketplace.

  • Mobile: iPhone 5 [320x568px]
  • Mobile: iPhone 6 [375x667px]
  • Mobile: iPhone 6 plus [414x736px]
  • Tablet: iPad 1/2/3/4/mini Landscape [1024x768px]
  • Tablet: iPad 1/2/3/4/mini Portrait [768x1024px]
  • Desktop: Macbook Air 11" [1366x768px]
  • Desktop XL: Macbook Air 13" [1440x900px]
  • Desktop XL: Macbook Pro 15" [1920x1200px]
  • Desktop XL: Thunderbolt display 27" [2560x1440px]

When reporting a bug that is design related - please reference one of these screen sizes in the ticket.

To view the site at the above screen sizes - inspect element in Chrome and then use the mobile tool to resize the browser based on that device.
